## Findings

### Finding 1 — Test-Isolation-Bug in nl-history-live.test.ts — **resolved (e68594e)**
- **Severity:** Low
- **Wann entdeckt:** Erster RLS-Suite-Run um 11:40 UTC
- **Symptom:** Test "3 INSERTs -> 3 Rows DESC" fetched 6 Rows statt 3
- **Root-Cause:** `fetchHistoryViaPg(10)` ohne Owner-Filter holt ALLE `automation_rule.sculpt_attempt`-Rows. Pre-existing 3 echte Production-Audit-Rows (SLC-753 + SLC-755 Live-Smokes mit qa-admin actor_id, NICHT durch RPT-460-Cleanup erfasst weil dieser nur `created_at >= 2026-05-18 09:50 UTC` filter) plus 3 Test-Inserts = 6 Rows.
- **Code-Bug-Status:** **Kein Code-Regression**. `parseHistoryRow` ist korrekt (durch RPT-459 9/9 ACs Live-Smoke verifiziert). Reines Test-Design-Problem.
- **Fix:** Commit `e68594e` — `fetchHistoryViaPg` akzeptiert jetzt `string | string[]` als ownerScope (mit `actor_id = ANY($2::uuid[])` IN-Clause). Test-1 filtert auf `[TEST_USER_A, TEST_USER_B]`. Tests 2-4 unveraendert (nutzten schon Single-Owner-Filter). Pollution-resistant fuer alle V7.6+ Live-Smokes.
- **Verifikation:** Re-Run der RLS-Suite nach Push: 146/146 PASS.

### Finding 2 — Production-Outage ISSUE-071-Repeat — **resolved (Workaround)**
- **Severity:** High (operational, transient)
- **Wann entdeckt:** Beim ersten RLS-Suite-Run um 11:37 UTC
- **Symptom:** RLS-Tests "the database system is not accepting connections" + "Connection terminated unexpectedly". DB-Logs: `PANIC: could not write to file pg_logical/replorigin_checkpoint.tmp: No space left on device`. Postgres-Crash-Loop seit 11:27 UTC. Disk 100% (38G/38G, 0 Avail).
- **Root-Cause:** Disk-Voll, Docker Build-Cache + unbenutzte Images akkumuliert seit letztem Cleanup (2026-05-16, RPT-429). ~30h Akkumulation, dann Crash. Exakt der ISSUE-071-Pattern (3. Auftritt: 2026-05-14, 2026-05-16, 2026-05-18).
- **Workaround:** Auf User-OK (AskUserQuestion 2026-05-18 11:39 UTC) ausgefuehrt:
  ```
  docker builder prune -af   # reclaim 5.695GB
  docker image prune -af     # reclaim 2.305GB
  ```
  Disk 100% → 82% (29G used, 6.8GB free). Postgres `pg_is_in_recovery=false` nach ~5s. Container `app` weiterhin healthy (Live-Smoke 200/307 OK).
- **Permanent Fix:** ISSUE-071 Next-Action #1 — Coolify/Hetzner-Host weekly Cron einrichten. **User-Pflicht-Action, noch offen.**
