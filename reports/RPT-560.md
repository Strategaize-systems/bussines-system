## Findings by Severity

### Blocker (resolved in-/qa via Deviation Rule 1)

**B-1 ã `free-question.ts` loggte unredacted `question` ins audit_log**

- **Pfad:** `cockpit/src/lib/ki-workspace/free-question.ts:122`
- **Symptom:** `logIsKnowledgeQuery({ ..., queryExcerpt: question, ... })` mit `question` = original User-Input ohne PII-Redact.
- **Spec-Verletzung:**
  - **AC-871-4:** "Frage `test@example.com Vollmacht-Klausel` wird VOR IS-Call zu `[email] Vollmacht-Klausel` redacted" ã fuer den Wire-Call wird das vom Adapter korrekt gemacht, aber der **Audit-Pfad** ist eine separate Datensenke, die ebenfalls redact-ready Daten braucht.
  - **AC-871-8 / DEC-258:** `changes_after.query_excerpt` ist explizit als "PII-redacted, max 200 chars" spezifiziert.
  - **Live-Smoke Pfad 4:** `qa/SLC-871-live-smoke.md:82-93` erwartet "audit_log query_excerpt enthaelt `[email]` und `[phone]`, NICHT die Original-Werte." ã wuerde reproduzierbar fehlschlagen.
- **Wurzel:** Pre-Fix-Kommentar im Code argumentierte "PII ist via redactPiiFromQ bereits Adapter-intern entfernt worden" ã falsche Schlussfolgerung. Der Adapter redacted nur die Wire-Query (URL-Param `q`), der Audit-Pfad bekommt das Original.
- **Risiken:** DSGVO-Verletzung. Im audit_log landen Mandanten-E-Mails/Telefonnummern unredacted und sind dauerhaft persistiert. Trotz "Internal-Test-Mode" der Production-Realitaet ein hartes Foul.
- **Fix (Deviation Rule 1 ã Auto-Bug-Fix):** Import `redactPiiFromQ` ergaenzt, im Audit-Aufruf `queryExcerpt: redactPiiFromQ(question)`. Kommentar ueberarbeitet auf den richtigen Grund ("Adapter redacted nur die Wire-Query, der Audit-Pfad ist eine separate Datensenke").
- **Test-Coverage:** Eine eigene Vitest-Datei fuer free-question.ts wurde NICHT geschrieben ã der Server-Action haengt an 5+ Side-Effects (Bedrock + Supabase + audit + IS-Adapter + Soft-Cap), Integration-Test-Setup haette Zeit-Box gesprengt. Die Komposition `logIsKnowledgeQuery({ queryExcerpt: redactPiiFromQ(question) })` ist trivial-visuell-verifizierbar. `redactPiiFromQ` ist 8 Cases abgedeckt. Live-Smoke Pfad 4 ist die kanonische End-to-End-Validation. Pre-Live-Pflicht.
- **Status:** **RESOLVED in-/qa.** Erkannt + Gefixt + Quality-Gates re-validiert (1204/1204 unchanged).
