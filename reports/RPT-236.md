## End-to-End-Workflow-Smoke (User-Live-Smoke-Plan)

Der durchgaengige Workflow vom Branding-Setup bis zur fertigen Mail. Sollte ca. 5-10 Min dauern. Erst danach kann V5.3 als release-ready bezeichnet werden.

### Setup (einmalig, falls noch nicht passiert)

1. Login auf [https://business.strategaizetransition.com](https://business.strategaizetransition.com)
2. `/settings/branding` aufrufen — Branding-Daten + Logo + Farben sind bereits gesetzt (aus SLC-531 Live-Smoke). Falls nicht: Logo hochladen + Primaerfarbe setzen + speichern.

### Schritt 1 — Eigene Vorlage anlegen

1. `/settings` → Reiter "Templates" oder direkt im Composing-Studio "Neue Vorlage"-Button (haengt von UI-Pfad ab)
2. **Manuell:** Titel "QA-Test Vorlage", Subject "Test {{firma}}", Body mit `{{vorname}}` + `{{firma}}` + `{{deal}}`-Variable
3. ODER **per KI-Generator:** Anweisung "Erstellt eine kurze Vorlage fuer eine Erstansprache an einen Steuerberater" → "Generieren" → Felder werden befuellt → User editiert nach Bedarf → "Speichern"

**Pruefkriterien Schritt 1:**
- [ ] Vorlage erscheint in der Liste mit Kategorie + Sprache
- [ ] Bei KI-Generierung: keine erfundenen Fakten/Namen, deutsche Sprache, Variablen sinnvoll eingesetzt
- [ ] Vorlage ist editierbar und speicherbar

### Schritt 2 — Composing-Studio mit Deal-Kontext oeffnen

1. Pipeline → einen bestehenden Deal oeffnen → "E-Mail"-Button klicken (deal-actions.tsx Einstiegspunkt)
2. Landung auf `/emails/compose?dealId=...&contactId=...&companyId=...`
3. ODER: Mein Tag → "E-Mail"-Button auf einer Aufgabe (mein-tag-client.tsx oder ki-workspace.tsx)
4. ODER: Focus → "E-Mail"-Button auf aktivem Item (focus-client.tsx)

**Pruefkriterien Schritt 2:**
- [ ] 3-Panel-Layout sichtbar (Templates links, Erfassen-Spalte mitte, Live-Preview rechts)
- [ ] Deal-Kontext-Banner zeigt Deal-Name + Kontakt + Firma
- [ ] Templates-Panel listet System+Custom-Vorlagen mit Filter
- [ ] Default-Filter "system" oder "alle" (aus BL-403)

### Schritt 3 — Vorlage anwenden + Variablen befuellen

1. Vorlage aus Schritt 1 anklicken → Body+Subject werden gefuellt
2. Variablen-Modal poppt auf (oder Inline-Felder, je nach UI-Pfad) → Variablen werden aus Deal-Kontext **automatisch vorbefuellt** ({{vorname}}, {{firma}}, {{deal}})
3. User kann Variablen editieren oder leer lassen

**Pruefkriterien Schritt 3:**
- [ ] Vorlage wird angewendet, Body sichtbar
- [ ] Variablen aus Deal-Kontext vorbefuellt (KI-Vorausfuellung aus SLC-533 Recipient-Suggest)
- [ ] An / Betreff koennen geaendert werden

### Schritt 4 — Live-Preview pruefen

1. Live-Preview-Panel rechts zeigt **gerenderte HTML-Mail** mit:
   - Logo aus Branding (SLC-531)
   - Primaerfarbe aus Branding
   - Footer mit Firmen-Name
   - Variablen ersetzt

**Pruefkriterien Schritt 4:**
- [ ] Iframe rendert ohne Console-Errors
- [ ] Logo sichtbar (kein broken-image — ISSUE-044 ist resolved)
- [ ] Primaerfarbe korrekt (z.B. Header-Akzent / Button-Farbe)
- [ ] Footer enthaelt Firmen-Name (z.B. "Strategaize Transition GmbH" — falls "Tnasition" steht: bekannter Daten-Korrektur-Punkt aus Next-Steps)
- [ ] Body wird mit ~250ms Verzoegerung aktualisiert wenn man im Erfassen-Editor tippt
- [ ] Variablen `{{vorname}}` etc. sind ersetzt

### Schritt 5 — KI-Improve testen (Korrektur / Formaler / Kuerzen)

1. Body editieren (z.B. Tippfehler einbauen)
2. KI-Improve-Bar im compose-form: "Korrektur" klicken
3. KI-Output ersetzt Body
4. Live-Preview aktualisiert sich

**Pruefkriterien Schritt 5:**
- [ ] KI-Output ist plausibel (Tippfehler korrigiert, Inhalt erhalten)
- [ ] Keine erfundenen Inhalte
- [ ] Live-Preview rendert neu
- [ ] Auch mit "Formaler" + "Kuerzen" testen

### Schritt 6 — Inline-Edit-Diktat testen

1. Inline-Diktat-Button klicken → Modal oeffnet
2. Voice-Aufnahme ODER Textarea: "Nach dem ersten Satz folgendes einbauen: Ich habe die Unterlagen vorbereitet."
3. "Vorschau erstellen" → KI-Diff zeigt minimale Aenderung
4. "Akzeptieren" → Body in Live-Preview entsprechend aktualisiert

**Pruefkriterien Schritt 6:**
- [ ] Modal oeffnet, Voice-Recording funktioniert
- [ ] Diff-Vorschau zeigt nur die geaenderte Stelle
- [ ] Akzeptieren propagiert in Live-Preview
- [ ] Verwerfen laesst Body unveraendert

### Schritt 7 — Senden + Follow-up-Task

1. Empfaenger-Adresse eintragen (eigene Mail-Adresse fuer Test)
2. Follow-up-Datum setzen (optional)
3. "Senden"-Button klicken
4. Loading-State + Success-Redirect zu /emails

**Pruefkriterien Schritt 7:**
- [ ] Mail wird tatsaechlich gesendet (in eigenem Postfach pruefen)
- [ ] Mail-Render im Postfach zeigt Branding korrekt (Logo + Farbe + Footer)
- [ ] Follow-up-Task wurde erstellt (in /aufgaben pruefen)
- [ ] /emails zeigt die gesendete Mail in der Liste mit Tracking-Indikatoren
- [ ] Outlook-Test (falls Eigentum): Mail sieht in Outlook nicht zerbrochen aus (SLC-531 Outlook-Smoke war noch offen)

### Schritt 8 — Cross-System-Konsistenz

1. Deal-Workspace wieder oeffnen → Aktivitaeten-Timeline zeigt die gesendete Mail
2. Kontakt-Workspace → E-Mail-Tab zeigt sie ebenfalls
3. Mein Tag → Follow-up-Task taucht auf (falls Datum heute oder morgen)

**Pruefkriterien Schritt 8:**
- [ ] Mail-Activity verlinkt korrekt zu Deal + Kontakt + Firma
- [ ] Follow-up-Task ist mit Deal verknuepft
- [ ] Tracking-Pixel funktioniert (Open-Indicator, falls Empfaenger Mail oeffnet — kann etwas dauern)
