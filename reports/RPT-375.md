## 2. Root-Cause-Analyse — Hotfix-1 war nicht ausreichend

RPT-374 hatte ISSUE-059+060 mit 3 Patterns diagnostiziert:
1. MeetingSheet als direct child of DropdownMenuContent (returnt intern `<Sheet>`)
2. `<div className="md:hidden">` Wrapper um DropdownMenuItem
3. EnrollButton als direct child mit `<span onClick>` Wrapper

Hotfix-1 (`26a4627`) hat alle drei Patterns korrigiert — Build, 515/515 Tests, Lint-Diff
clean. Aber Live-Smoke nach Coolify-Deploy zeigte: BEIDE Dropdowns crashen weiter mit
genau denselben Errors (Base UI #31 + React #418).

Erweiterte Diagnose: Bundle-Hash-Vergleich (`0hfw~zchjv985.js` etc.) zwischen lokalem
Hotfix-Build und Production-Build identisch — Hotfix WAR live, aber nicht ursaechlich.

**Eigentliche Ursache:** `<DropdownMenuLabel>Meeting</DropdownMenuLabel>` bzw.
`<DropdownMenuLabel>Mehr Aktionen</DropdownMenuLabel>` sind direkte Children von
`<DropdownMenuContent>`. `DropdownMenuLabel` = Base UI `Menu.GroupLabel`. Base UI
verlangt **GroupLabel ausschliesslich innerhalb Menu.Group**. GroupLabel als
direkter Child von `Menu.Popup` triggert beim Open Base UI Error #31, der dann
die ganze Seite via Hydration-Cascade (React #418) crasht.

Build/Tests konnten das nicht erkennen — Crash entsteht erst beim Open-Render-Cycle,
nicht im SSR-Render.
