# RPT-415 — V7.1 Architecture

## 1. Architecture summary

V7.1 ist ein **Pure-Reuse-Sprint** auf V7-Stack:

- **Keine neue Auth-Surface:** `assertRole` + `read-only-context` + `SIDEBAR_CONFIG` aus V7 werden wiederverwendet.
- **Keine Schema-Migration:** `profiles.role`, `*.owner_user_id`, `audit_log` aus MIG-033/034 ausreichend.
- **Keine neue Provider/Service-Anbindung:** alles im bestehenden V7-Stack.
- **3 Features orthogonal:** FEAT-711 (Permission-Layer auf 11 Settings-Pages) + FEAT-712 (Drilldown-Vollausbau via PipelineView-Props-Erweiterung) + FEAT-713 (4 Defense-in-Depth-Guards + Doc-Sync).

**FEAT-711:** Mehrstufiger Schutz: Server-Side-Sidebar-Filter (Layout) → Settings-Layout-Sidebar (rollen-gefiltert) → Settings-Landing-Kacheln-Filter (rollen-gefiltert) → Direct-URL-`assertRole`-Guard. Defense-in-Depth.

**FEAT-713:** 4× `assertNotReadOnlyContext()` als first line in Mutate-Server-Actions. Greift heute via Layout-Wrap; ISSUE-066-AsyncLocalStorage-Gap bleibt V7.5-Mitigation.

- Permission-Cut nach User-Walkthrough-Risiko-Klasse (Admin = organisationsweit, Admin+Teamlead = team-operativ, Alle = persoenlich)

- Keine Aenderung an Audit-Trail (DEC-197)

## 8. Constraints & Tradeoffs

| Constraint | Implikation |
|---|---|
| Pattern-Reuse-First | PipelineView-API waechst um 2 optionale Props (akzeptabel) |
| Keine Schema-Migration | Audit-Trail-Erweiterung V7.2-Defer |
| Internal-Test-Mode bleibt aktiv | Compliance-Gate-Anforderungen bleiben Pre-Production-Thema |
| Sprint-Reihenfolge SLC-711 → 712 → 713 | SLC-713 kommt zuletzt; bei Zeit-Druck parallel zu 712 moeglich |
| Mein-Profil-Page existiert nicht | Member sieht in Settings nur Working-Hours + Meeting + Briefing; Stub V7.2+ falls Bedarf |
