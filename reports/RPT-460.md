Sonner-Library ist **im SSR-Chunk** vorhanden (`node_modules_sonner_dist_index_mjs_04op7k1._.js` in `.next/server/chunks/ssr/`), aber **nicht im client static chunks** (Tree-Shaking hat sie entfernt). `next-themes` ist ueberhaupt nicht traced.

### Sonner-Source-Analyse (sonner@2.0.7)
```javascript
// /app/node_modules/sonner/dist/index.mjs (gefunden in dev-host)
if (!filteredToasts.length) return null;
return React.createElement("ol", { "data-sonner-toaster": true, ... });
```

Sonner rendert das innere `<ol data-sonner-toaster>` nur bei `filteredToasts.length > 0`. Da im DOM nur die leere `<section>` da ist (= Sonner-Toaster ist mounted, hat aber keine Toasts) — und kein Console-Error feuert — bedeutet das: `toast.success()` updated **die ToastState-Singleton-Instance**, an die der Toaster subscribed ist, nicht. Wahrscheinlich Module-Duplication oder Hydration-Issue mit dem Wrapper.
