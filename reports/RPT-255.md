## Problems found

1. **Backend-Schema-Inkonsistenz `notes`-Feld** — Zod-Schema `proposalEditSchema` validiert `notes` als optional, aber DB-Spalte heisst `scope_notes` (oder existiert garnicht als `notes`). Frontend-Editor sendet `notes` NICHT, daher kein Laufzeit-Bug. Aber latentes Risiko, falls eine andere Action `notes` patcht. → Empfehlung: Backend-Cleanup im /qa-Schritt (`notes` aus Zod-Schema entfernen ODER DB-Spalte aliasen).

2. **MT-9 Pipeline-Kontextmenue Ort** — Spec sagt "Pipeline-Card-Kontextmenue auf `/pipeline/[slug]/page.tsx`". Ich habe stattdessen die `KanbanCard`-Komponente erweitert (zentraler Ort). Funktional aequivalent, Pfad anders dokumentiert. Slice-Spec-Drift: Files-to-Touch nennt `pipeline/[slug]/page.tsx`, gemacht in `components/kanban/kanban-card.tsx`.

3. **dnd-kit + Dropdown-Trigger Konflikt** — KanbanCard hat `{...listeners}` von dnd-kit auf dem Card-Element. Dropdown-Trigger braucht `onPointerDown={e => e.stopPropagation()}`, sonst startet jeder Klick auf MoreVertical einen Drag. Pattern dokumentiert, sollte fuer SLC-554 Status-Buttons im Workspace nicht relevant sein (keine dnd-listeners im Editor-Panel).

4. **Pre-existing Lint-Errors** — 221 Repo-Errors (V5.4-Carryover) bleiben unangefasst. Memory-Note: nicht meine.
