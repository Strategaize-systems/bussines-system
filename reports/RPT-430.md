## V7.2 Definition (Kurzfassung)

**Problem:** V7-Release hat 3 Test-Infra-Schwaechen als Accepted Risks liegengelassen — die `npm run test:rls`-Suite hat 96 SKIPPED + 0 Bulk-Reassign-Tests + manuell-anzulegender qa-admin pro /qa-Session. Blockiert keine V7.1-Funktion, untergraebt aber Multi-User-Confidence und wird in V7.5 NL-Automation (lebt im Multi-User-Kontext) zur Bremse.

**Goal:** `npm run test:all` 897 PASS (779 jsdom + 118 RLS), Seed-Script idempotent + in Container-Bootstrap, 3 Test-Rollen automatisch beim Coolify-Redeploy verfuegbar.

**V1-Scope (3 In-Scope-Items):**
1. **BL-471** create-qa-test-users.mjs um qa-admin (UUID `0...0ba001`) erweitern.
2. **BL-473 / ISSUE-073** seed:multi-user gegen Coolify-DB applicieren + Container-Bootstrap-Hook.
3. **BL-474 / ISSUE-074** vitest.rls.config.ts Path-Alias-Resolver via vite-tsconfig-paths.

**Out-of-Scope:** Kein Code-Touch an V7-Server-Actions/RLS-Policies/SQL-Helper, keine neuen Test-Cases, keine CI-Pipeline-Anbindung, keine V7.5-Vorgriffe, keine Schema-Migration.

**Constraints:**
- Coolify-DB ist Live-Production-Postgres (Internal-Test-Mode) — Seed nur INSERT ON CONFLICT DO NOTHING, kein DELETE/UPDATE auf Production-IDs.
- UUID-Range `0...0ba0xx` (qa-* Prefix) im Test-Team-77 — klar separiert vom Strategaize-Team.
- Container-Bootstrap darf App-Start nicht blockieren (Seed-Failure = Warning, kein Hard-Stop).
- Pattern-Reuse-Pflicht: Path-Alias-Resolver-Pattern aus jsdom-Default-vitest-Config 1:1 portieren.
- Time-Box **3-4h, max 1 Slice.** Bei Time-Box-Hit: Container-Bootstrap auf V7.3 verschieben, V7.2 reduziert auf Pure-Seed-Script + Manual-Apply.

**Success Criteria:**
- `npm run test:rls -- v7-rls-matrix` = 96 PASS
- `npm run test:rls -- bulk-reassign` = 7 Suites laden + laufen
- `npm run test:all` = 897 PASS, 0 FAIL, 0 SKIP
- psql `SELECT email,role FROM profiles WHERE email LIKE 'qa-%@strategaize.test'` = 3 Rows
- 2x seed:multi-user Idempotenz-PASS
- Coolify-Redeploy → Test-Daten automatisch da

**Delivery Mode:** internal-tool (unveraendert).
