### Medium

- **Latenter Schema-Bug `notes`-Feld in Zod-Schema** (resolved im QA-Lauf)
  Zod-Schema `proposalEditSchema` validiert `notes` als optional, aber DB-Tabelle `proposals` hat KEINE `notes`-Spalte (MIG-026 fuegt nur `valid_until`, `payment_terms`, `parent_proposal_id`, `accepted_at`, `rejected_at`, `expired_at`, `pdf_storage_path` hinzu). Existing-Spalte heisst `scope_notes`. Wenn jemand `updateProposal(id, { notes: "X" })` aufruft, wuerde Postgres mit "column 'notes' does not exist" fehlen. Aktuell ungenutzt (Frontend-Editor sendet kein `notes`), aber Schema-Drift = latentes Risiko.

  **Fix angewendet (Deviation-Rule 1):** `notes`-Feld aus `proposalEditSchema` entfernt; `ProposalUpdatePatch`-Type in `actions.ts` entsprechend bereinigt. Tests + Build erneut gruen.

- **MT-9 Pfad-Drift gegenueber Slice-Spec**
  Slice-Spec `Files to Touch` nennt `cockpit/src/app/(app)/pipeline/[slug]/page.tsx` als Ort fuer das Pipeline-Kontextmenue. Tatsaechlich implementiert in `cockpit/src/components/kanban/kanban-card.tsx` — zentralerer Ort, da `[slug]/page.tsx` nur `<KanbanBoard>` rendert und die Cards selbst in `kanban-card.tsx` leben. Funktional aequivalent, dokumentations-Drift.

  **Auswirkung:** keine Funktionalitaets-Auswirkung. RPT-255 dokumentiert das.

### Low

- **dnd-kit + Dropdown-Trigger Pattern**
  `KanbanCard` hat `{...listeners}` von dnd-kit auf dem Card-Element. Das `<DropdownMenuTrigger>` braucht `onPointerDown={(e) => e.stopPropagation()}` + `onClick={(e) => e.stopPropagation()}`, sonst startet jeder Klick auf den MoreVertical einen Drag. Pattern dokumentiert in kanban-card.tsx Zeile ~91-95. Sollte fuer SLC-554 Status-Buttons im Workspace nicht relevant sein (kein dnd im Editor-Panel).

- **Pre-existing Repo-Lint-Errors (V5.4-Carryover)**
  221 Errors aus V5.4-Sprint, nicht verursacht durch SLC-552. Per Memory `feedback_business_system_focus` und Slice-Scope-Discipline nicht angefasst.

- **Schnell-Edit-Pencil bleibt** (AC12 explizit)
  `/proposals`-Listing hat jetzt zwei Edit-Optionen pro Draft-Row: blauer "Bearbeiten"-Button (neuer Workspace) + grauer Pencil-Icon (Legacy V2-Modal). UX-Diskussion delegiert auf V5.5.x oder V5.6+ (Default: behalten).

## Open Risks / Missing Checks

- **Reorder-Race-Condition** — wenn User waehrend laufendem `reorderProposalItems` weitere Edits macht, koennte UI-State inkonsistent werden. Mitigation: `pointer-events-none` + `opacity-70` waehrend `reordering=true`. Akzeptierter Tradeoff (Single-User-Single-Tab-Annahme aus Slice-Risiken).
