## Post-Deploy-Fixes (3)

### Fix 1: Profile-Row fehlte (FK-Constraint)
- Problem: Auth-User `96322a0a...` hatte keine Row in `profiles`. `user_settings`-Upsert schlug fehl mit FK-Violation.
- Ursache: Historisch — User wurde in auth.users angelegt, aber kein Trigger/Script erzeugte eine profiles-Row.
- Fix: `INSERT INTO profiles (id, display_name, role) VALUES ('96322a0a...', 'Richard Bellaerts', 'admin')` auf Hetzner.
- Impact: Settings/Meetings funktioniert jetzt.
- Praevention: Fuer kuenftige User-Anlage sicherstellen, dass profiles-Row synchron erzeugt wird (auth.users trigger oder Signup-Flow).

### Fix 2: Coolify Cron Container-Name
- Problem: Coolify Scheduled Tasks hatten keinen Container-Namen → "More than one container exists but no container name was provided".
- Fix: Alle Scheduled Tasks auf Container `app` gesetzt (Coolify UI).
- Impact: Alle Crons laufen jetzt korrekt.

### Fix 3: Settings Padding
- Problem: Settings + Settings/Meetings hatten kein `px-8 py-8` (alle anderen Seiten schon).
- Fix: Commit fa5f5d6, wird beim naechsten Redeploy aktiv.
- Impact: Rein visuell — konsistentes Layout.
