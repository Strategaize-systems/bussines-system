## Rollback Path

V4.2 ist **rein additiv** — kein Impact auf bestehende Tabellen, Features oder Workflows.

**Code-Rollback:**
- Docker Image Rollback via Coolify auf V4.1 Image
- Wissen-Tab verschwindet, alle anderen Features bleiben

**Schema-Rollback:**
```sql
DROP FUNCTION IF EXISTS search_knowledge_chunks;
DROP TABLE IF EXISTS knowledge_chunks;
DROP EXTENSION IF EXISTS vector;
```

**Risiko des Rollback:** Gering. Einziger Verlust: gespeicherte Embeddings. Re-Embedding nach erneutem Deploy jederzeit moeglich (Backfill ist idempotent).

## Feature Isolation

Der Wissen-Tab ist ein **neuer Tab** im Deal-Workspace. Er beruehrt keine bestehenden Tabs (Uebersicht, E-Mails, Meetings, Dokumente, Aktivitaeten). Bei Fehler im Wissen-Tab sind alle anderen Deal-Workspace-Funktionen unbetroffen.

Auto-Embedding Trigger laufen als fire-and-forget mit try/catch. Wenn Embedding fehlschlaegt, laeuft der Parent-Prozess (Meeting-Summary, IMAP-Sync, Activity-Create, Document-Upload) ungestoert weiter.
