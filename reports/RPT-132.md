## Supautils PG15 Blocker — neuer Deploy-Blocker dokumentiert

`CREATE EXTENSION IF NOT EXISTS vector` schlug auf Self-hosted Supabase fehl: `permission denied for function pg_read_file`. Ursache: supautils wird per `session_preload_libraries` geladen und intercepted alle CREATE EXTENSION Calls. Der before-create Hook ruft `pg_read_file` in einem PG15 security-restricted context auf, wo die Funktion blockiert ist.

**Workaround (5 Schritte, ~2 Minuten):**
1. `postgresql.auto.conf` schreiben mit `session_preload_libraries = ''`
2. `docker restart <db-container>`
3. `CREATE EXTENSION IF NOT EXISTS vector;`
4. `postgresql.auto.conf` zuruecksetzen (nur Header)
5. `docker restart <db-container>`

IMP-084 im Dev System aktualisiert mit dem vereinfachten Workaround.
