# Discovery: Business System V3 — Operative Kontextlogik

## Raw Idea Summary

Das Business System soll von einem feature-zentrierten CRM mit gleichberechtigten Listen-Seiten zu einem kontextzentrierten Business-Development-Betriebssystem umgebaut werden. Die klassischen CRM-Bereiche bleiben als Fallback, aber die primäre Arbeitslogik verschiebt sich auf Workspaces, KI-unterstützte Tagessteuerung, Analyse-Cockpits und eine Wissensschicht mit Review-Mechanik.

## Likely Main Goal

Das System soll nicht mehr über Feature-Seiten gesteuert werden, sondern über Kontext und Prozess. Der User soll morgens "Mein Tag" öffnen und kontextgesteuert arbeiten: in Deals einsteigen, KI-Briefings lesen, Aktionen ausführen, Meetings vorbereiten.

## 5-Schichten-Zielstruktur

1. **Operative Arbeit:** Mein Tag, Deal-/Firmen-/Kontakt-Workspaces, Aufgaben, Termine, E-Mails, KI-Unterstützung
2. **Analyse:** Persönliches + Management-Cockpit, KI-Datenabfragen, KPI-/Trendansichten
3. **Prozesslogik:** Phasen, Soll-Schritte, Stagnation, nächste Aktion, Folgeaktivitäten
4. **Wissens-/Insight-Schicht:** Wissensbasis, Insight-Review-Queue, Einwände, Best Practices
5. **Fallback/Verwaltung:** Listenansichten, Stammdatenpflege, Exception Queue, Settings

## Gap-Analyse Zusammenfassung

### Kann übernommen werden
- Datenmodell Core (15 Tabellen)
- Pipeline-Logik (3 Pipelines, Stage-Transitions, Required-Fields, Auto-Status)
- SMTP E-Mail + Follow-up-Tracking
- Whisper Voice-Integration
- Supabase Auth + Middleware
- UI-Komponentenbibliothek (shadcn/ui, FilterBar, KPICard, KanbanBoard)
- Fit-Assessment-Engine

### Muss umgebaut werden
- DealDetailSheet → Deal-Workspace (eigene Seite)
- Company/Contact Detail → Workspaces mit Deal-Kontext + KI
- Dashboard → Management-Cockpit mit LLM-Layer
- Mein Tag → echte Kalender-Daten, Meeting-Prep, KI-Backend
- Navigation → hierarchische Schicht-Struktur
- Profiles → Rollen, Team
- RLS → rollenbasiert

### Komplett neu
- Wissensobjekte (5 neue Tabellen, Phase 3+)
- Steuerungsobjekte (5 neue Tabellen, Phase 2+)
- Kalender-Objekt
- LLM-Integration Layer (Bedrock)
- Prozesslogik-Engine
- Persönliches Analyse-Cockpit
- Wissensbasis + Insight-Review-Queue
- Exception Queue
