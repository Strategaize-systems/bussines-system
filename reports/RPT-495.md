## Wiring Verification Details

```
Request to /any-app-page
   ↓
1. layout.tsx (Server-Component)
   ├── const profile = await getProfile()
   ├── const teamSize = await getTeamSize(profile)
   │     ├── if profile.team_id === null → return 1 (no DB call)
   │     └── else → SELECT count(id) FROM profiles WHERE team_id = X
   ├── <MobileLayoutShell role={profile.role} hideTeamSection={teamSize === 1}>
   ↓
2. mobile-layout-shell.tsx (Client-Component, "use client")
   ├── Empfaengt: role, hideTeamSection (Default false), children
   ├── Sidebar Desktop: <Sidebar role variant="desktop" hideTeamSection />
   ├── Sidebar Mobile-Drawer: <Sidebar role variant="mobile" hideTeamSection onItemClick />
   ↓
3. sidebar.tsx (Client-Component)
   ├── Empfaengt: role, hideTeamSection (Default false), variant, onItemClick
   ├── const topGroups = useMemo(
   │     () => groupWithSubGroups(filterByRole(role, { hideTeamSection })),
   │     [role, hideTeamSection],
   │   )
   ↓
4. sidebar-config.ts (Pure-Function)
   ├── filterByRole(role, opts?)
   │   ├── filter SIDEBAR_CONFIG by visibleFor (Permission-Filter)
   │   ├── if opts?.hideTeamSection: additional filter section !== "TEAM"
   │   └── return SidebarItem[]
   ↓
5. Sidebar rendert Topgroups → Sidebar zeigt KEINE TEAM-Section bei Solo
```

**Type-Safety:** Alle 4 Layer durch TypeScript-Compile-Check abgesichert. Build clean.

**No-Flash-Garantie:** `hideTeamSection`-Prop ist server-determined (`teamSize === 1`). Initial SSR-Render setzt bereits korrekten Filter. Client-Hydration nutzt gleiche Props → kein Re-Filter, kein Flash.
