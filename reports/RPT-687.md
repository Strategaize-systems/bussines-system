## Finding — ISSUE-145 (High, gefunden im Smoke, sofort behoben)

**Symptom (echter UI-Flow, IONOS-Mail `3523b489…`):** Block-State korrekt (srcDoc + Banner „Externe Bilder blockiert [Bilder laden]", Bilder gestrippt). Nach Klick „Bilder laden" wechselte der iframe auf `src=/api/emails/[id]/body`, blieb aber leer (Höhe 80px, `contentDocument` null), die Mail verschwand, Bilder rendern nie. Console: **„Framing '…' violates frame-ancestors 'none'"**.

**Root Cause:** `next.config.ts` legte die globalen SECURITY_HEADERS (CSP `frame-ancestors 'none'` + `img-src 'self' data: blob:` + `X-Frame-Options: DENY`) via `source: "/:path*"` auf ALLE Routen inkl. `/api/emails/[id]/body`. Next.js `headers()`-Config **gewinnt** gegen die im Route-Handler gesetzte `BODY_CSP` → die Response trug die globale CSP. Bestätigt per authentifiziertem Fetch (Response-CSP = global) UND Live-Framing-Violation. `next build`/tsc/eslint/76 Unit-Tests alle grün — nur im Feature-Flow-Smoke sichtbar.

**Fix (2 chirurgische Änderungen):**
1. `cockpit/next.config.ts` — `source` auf Negative-Lookahead `"/((?!api/emails/[^/]+/body).*)"`: die globalen Header überspringen genau diese eine Route.
2. `cockpit/src/app/api/emails/[id]/body/route.ts` — `BODY_CSP` um `frame-ancestors 'self'` ergänzt (same-origin-Einbettung erlaubt, cross-origin verboten — ersetzt das global entfernte `X-Frame-Options: DENY` als Defense-in-Depth).

Verifikation code-side: tsc 0, 76 Unit-Tests (sanitize+iframe+csp) grün, `next build` 0 (Negative-Lookahead-Source valide).
