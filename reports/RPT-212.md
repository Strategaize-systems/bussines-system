## Inline-Fix angewendet (per Deviation Rule 1)

**Issue:** AC9 verlangte "Liste verfuegbarer Tokens **mit Beispiel-Wert**". Initial-Implementierung in ComplianceTemplateBlock.tsx zeigte nur `name + description`, nicht aber `example`. Token-Daten waren in `tokens.ts` korrekt vorhanden, nur das UI-Render-Statement war unvollstaendig.

**Fix:** ComplianceTemplateBlock.tsx Z. 120-138: Token-Liste umgebaut zu zwei Zeilen pro Token (Name+Description oben, "Beispiel: <example>" als kleine zweite Zeile darunter). Layout bleibt 2-spaltig auf sm+, ist auf mobile single-column. Klassifikation als "auto-fix bug" per `deviation.md` Rule 1 — kein Architektur-Change, kein API-Change, nur UI-Content-Gap.

## Open Risks / Missing Checks

- **Browser-Verifikation**: Save-Round-Trip, Reset, Copy-to-Clipboard in echtem Chrome sind nicht im QA-Lauf machbar (kein Browser-MCP, keine echte Auth-Session). Diese 3 Punkte muessen User selbst testen. AC5/AC6/AC7/AC8 sind code-seitig verifiziert (DB-Layer + Wiring + Test-Engine), aber Browser-Behavior kann nur Live verifiziert werden.
- **Performance/Caching**: `revalidatePath("/settings/compliance")` nach jeder Mutation — bei haeufiger Edit-Sequenz kein Issue (3 Templates, kleine Payload). Kein Last-Test im QA-Scope.
- **Cross-Browser Clipboard**: `navigator.clipboard.writeText` braucht Secure Context (HTTPS). Production laeuft hinter Coolify-TLS — funktioniert. In Dev (HTTP localhost) faellt es bei Browser-Sicherheitseinstellungen ggf. zurueck auf den Catch-Branch ("Kopieren fehlgeschlagen — bitte manuell markieren") — ist sauber gehandelt.
