## 4. Issues found

### NEU: ISSUE-057 — FollowupEngine.openProposals query bricht auf nicht-existenter Spalte `proposals.value`

- **Severity:** Medium
- **Status:** open
- **Evidence:** `[FollowupEngine] Open proposals query failed: column proposals.value does not exist` 3x in 16h Container-Logs
- **Code-Stelle:** `cockpit/src/lib/ai/followup-engine.ts:194-208` selektiert + sortiert auf `value`, das Feld existiert in der Tabelle nicht. Tatsaechliche Spalten sind `subtotal_net` / `tax_amount` / `total_gross`.
- **Schema-Historie:** SLC-405 (V4) hat den Engine-Code gegen ein damaliges `proposals.value`-Feld geschrieben. SLC-551 / MIG-026 (V5.5, 2026-04-29) hat das Proposal-Schema komplett restrukturiert (Snapshot-Pattern, price_at_creation, separates `proposal_items`-Table). Die neue Schema-Variante kennt kein `value` mehr.
- **Latent seit:** V5.5 Release 2026-05-01 (REL-020) — 6 Tage. Bisher unbemerkt, weil Cron auf `errors=0` reportet (Code-Pfad `catch` setzt failed-Counter nicht inkrementell), und weil aktuell 0 echte Kandidaten existieren.
- **User-Impact aktuell:** Null — keine Followup-Kandidaten in der DB.
- **User-Impact zukuenftig:** Sobald reale Proposals versendet werden, wuerden KI-Wiedervorlagen fuer Open-Proposals stillschweigend uebersprungen (Stagnant-Deal-Pfad funktioniert weiter).
- **Workaround:** Keiner — Code-Fix erforderlich.
- **Empfohlener Fix:** `value` → `total_gross` an 2 Stellen (Select-String Z. 199 + order-Spalte Z. 207). Klein genug fuer einen Hotfix-Slice (~15min Code + Vitest-Patch + Live-Smoke).
