## Problems Found

1. **Cal.com Image-Version**: `calcom/cal.com:latest` verwendet — fuer Produktion sollte eine spezifische Version gepinnt werden. Nach erstem Start auf Hetzner die laufende Version notieren und pinnen.
2. **Coolify-Konfiguration noch offen**: Cal.com braucht eine eigene Domain-Routing-Regel in Coolify (cal.strategaizetransition.com → calcom:3000). Muss manuell in Coolify UI konfiguriert werden.
3. **Cal.com API-Key**: Wird erst nach erstem Start in der Cal.com Admin-UI generiert. CALCOM_API_KEY muss danach in Coolify Env-Vars gesetzt werden.
4. **Cal.com Webhook**: Muss nach Setup in Cal.com Admin unter Settings → Webhooks konfiguriert werden (URL: https://business.strategaizetransition.com/api/webhooks/calcom, Secret: CALCOM_WEBHOOK_SECRET).
5. **CALCOM_BOOKING_PATH**: Muss nach Cal.com Setup mit echtem Username/Event-Slug befuellt werden (z.B. `/rbellaerts/30min`).

## Open Points / Deferred Decisions

1. **Cron-Frequenz fuer calcom-sync**: Noch kein Coolify Scheduled Task angelegt. Empfehlung: taeglich 1x oder bei Bedarf manuell. Webhook-basierter Sync ist der primaere Sync-Mechanismus.
2. **Cal.com Admin-Setup**: Nach Deploy muss Admin-Account erstellt, Event-Types konfiguriert und Webhook eingerichtet werden — nicht automatisierbar.
3. **Jitsi-Integration**: Laut DEC-031 soll Jitsi fuer Video-Meetings kommen (V4.1). Cal.com kann Jitsi als Location-Provider nutzen — Konfiguration nach Jitsi-Setup.
