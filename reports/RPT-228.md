## Problems found

### F-1 (Slice-Spec-Drift, mit Deviation Rule 1 korrigiert): MT-5 Spalten-Bezeichnungen

**Befund:** Die Slice-Spec MT-5 referenziert `deals.primary_contact_id` und `emails.direction='inbound'`. Die tatsaechlichen DB-Spalten sind `deals.contact_id` (V1-Schema in `01_schema.sql`) und die Inbound-IMAP-Mails liegen in `email_messages` (V4, ohne `direction`-Spalte — alle Eintraege dort sind per Definition Inbound, weil IMAP-Sync). Die Legacy-`emails`-Tabelle (V2) hat zwar `direction`, ist aber primaer fuer Outbound `sendEmailWithTracking`-Writes da.

**Impact:** Hoch — eine 1:1-Umsetzung der Spec haette einen 500-Fehler erzeugt (Spalte existiert nicht).

**Fix (Rule 1 — Bug-Fix):** `recipient-suggest.ts` nutzt `deals.contact_id` und `email_messages` (sortiert nach `received_at DESC`). Slice-Spec dokumentiert wortlaut-getreue Intention; Implementation deckt sie auf der korrekten Schema-Realitaet ab.

**Severity:** Medium (verhindert AC4-Funktionalitaet ohne Fix; mit Fix gruen).

### F-2 (Verifikations-Gap): Browser-Render nicht direkt geprueft

**Befund:** Build-success ist nicht gleich Render-success. Mobile-Tab-Layout-Pixel, Voice-Permission-Flow, Live-KI-Improve-Latenz, Modal-Backdrop-Verhalten und Tab-State-Persistenz wurden nicht im Browser direkt verifiziert.

**Impact:** Niedrig — alle Patterns sind aus bereits live laufendem Code (`email-compose.tsx`, `voice-record-button.tsx`, `dialog.tsx`-Wrapper) uebernommen. Regression-Risiko ist begrenzt.

**Severity:** Low (typisches /frontend → /qa-Setup; Verifikation gehoert in /qa SLC-533).

**Status:** Wird in /qa SLC-533 explizit getestet.

### F-3 (Build-Side-Effect, NICHT von SLC-533): `[insights] fetch failed` waehrend Build

**Befund:** Beim `npm run build` erscheint `[insights] Failed to load pending insights: TypeError: fetch failed`. Verursacht von `/dashboard` Static-Generation-Phase, die ohne laufenden Supabase-Endpunkt einen Fetch versucht.

**Impact:** Keiner fuer SLC-533 — Build wird trotzdem mit Compiled successfully + 60/60 Routes generiert.

**Severity:** None (vorbestehend, V5+ Build-Time-Dashboard-Refresh-Pattern, kein neues Problem).
