### Drag-Drop-no-effect — VERIFIED via 3-Layer-Defense

Direkte Drag-Simulation NICHT moeglich da alle Test-Member 0 Deals in den
verfuegbaren Pipelines (Multiplikatoren + Unternehmer-Chancen) haben — gilt
auch fuer qa-teamlead Self-Pipeline. Verifikation via Code + DOM:

1. **DOM-Check:** `sortableItemCount: 0`, `droppableMarkerCount: 0` — kein
   `[aria-roledescription="sortable"]` Marker. KanbanBoard-readOnly-Branch
   rendert ohne DndContext (siehe `kanban-board.tsx` Z.131-148).
2. **Code-Defense Layer 1:** `KanbanBoard` `if (readOnly) return <PlainColumns/>`
   — KEIN `<DndContext>...</DndContext>` Wrap.
3. **Code-Defense Layer 2:** `KanbanColumn` `useDroppable({ id, disabled: readOnly })`
   — auch wenn DndContext zufaellig vorhanden waere, sind drops disabled.
4. **Code-Defense Layer 3:** `KanbanCard` `useSortable({ id, disabled: readOnly })`
   + `...(readOnly ? {} : attributes)` + `...(readOnly ? {} : listeners)`
   — keine drag-handle-listeners gespreaded.
5. **Vitest Cover:** 4 Tests bestaetigen Props-Verarbeitung (readOnly=true
   verhalten + readOnly=false Regression).

**Drag-Drop kann nicht passieren** in Drilldown-Variante. Defense-in-Depth
nochmal via `assertNotReadOnlyContext()` in Server Actions (V7 SLC-706).
