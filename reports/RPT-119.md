## Problems found

1. **Docker-Compose-Validation nur syntaktisch.** Lokaler `docker compose config --quiet` laeuft sauber (Exit 0), aber die tatsaechliche Jitsi-Stack-Integritaet (Prosody<->Jicofo<->JVB<->Jibri XMPP-Handshake) ist erst nach Coolify-Deploy auf Hetzner messbar. Risk-Matrix V4.1 nennt "Jibri-Recording schlaegt fehl (Chrome crasht)" als Mittel-Risiko — Mitigation bleibt Restart-Policy.
2. **Hetzner-Cloud-Firewall UDP/10000 nicht SSH-pruefbar.** UFW ist auf dem Server inactive (erwartet, Hetzner Cloud Firewall laeuft auf Hypervisor-Ebene). Die Regel laut STATE.md DONE, aber muss vor Smoke-Test in Hetzner Cloud Console visuell bestaetigt werden.
3. **Smoke-Test-Helper als .mjs statt .ts.** Slice-Spec MT-5 nannte `scripts/gen-test-jwt.ts`. Ich habe `.mjs` gewaehlt, um kein ts-node/tsx am Repo-Root zu benoetigen (Cockpit hat tsx intern, aber Script lebt auf Root-Ebene). Funktional identisch, Aufwand/Kosten-Nutzen spricht fuer `.mjs`.
4. **env vars_business.txt war bisher nicht gitignored** — Datei enthielt produktive Supabase-Keys und POSTGRES_PASSWORD. War nur "untracked", nicht geschuetzt. Jetzt via .gitignore abgesichert. Historisch kein Commit passiert (git log verifiziert), aber kein Glueck ist kein Prinzip. Als IMP-Kandidat fuer Dev-System notieren: "Neue Repo-Bootstrap sollte `env vars_*.txt` Pattern von Anfang an gitignoren".
