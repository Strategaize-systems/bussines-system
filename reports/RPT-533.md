## Discovery

User-Browser-Smoke 2026-05-23 ~14:05 UTC im Composing-Studio (`/emails/compose`). User schickte einen Test-Send mit angewendeter Template ("Danke nach Termin"), roter Banner ueber dem Senden-Button zeigte:

```
Could not find the 'owner_user_id' column of 'emails' in the schema cache
```

User-Screenshot in der Smoke-Session geteilt.

## Root Cause

`emails`-Tabelle (outgoing-Mail) hat **KEINE** `owner_user_id`-Spalte in Production-DB.

**MIG-033 (V7 SLC-704, 2026-05-14)** hat `owner_user_id` auf 8 Kerntabellen ergaenzt:
1. companies
2. contacts
3. deals
4. activities
5. meetings
6. proposals
7. **email_messages** (incoming)
8. calls

**Die outgoing-Mail-Tabelle `emails` wurde dabei vergessen.** Separate Tabelle von `email_messages` (incoming) — der gleiche Name ist verwirrend.

Code in `cockpit/src/lib/email/send.ts` Zeilen 133+232 inserted aber `owner_user_id`:
```typescript
.insert({
  ...(params.ownerUserId ? { owner_user_id: params.ownerUserId } : {}),
  ...
})
```

**Pre-V8.4-Verhalten:** Compose-Form-Caller (sendComposedEmail) setzte `ownerUserId` NICHT. INSERT-Spread `...({} : {})` liess die Spalte einfach weg. DB akzeptierte INSERT.

**V8.4-Verhalten (seit SLC-846, Deploy 09:18 UTC):** SLC-846 ergaenzte sendComposedEmail um `ownerUserId` (Pflicht-Param fuer tenantSlug-Lookup im DSE-Footer-Render). INSERT-Spread setzt jetzt die Spalte → DB rejects → Mail geht nicht raus.

## Severity-Bewertung

**BLOCKER.** Alle Mail-Sends aus dem Composing-Studio (Compose-Form, KI-Verbesserung) failed seit V8.4-Deploy 09:18 UTC bis MIG-040-Apply 14:08 UTC = **~5h Production-Downtime fuer outgoing-Mails via UI**.

Nicht betroffen: Cadence-Mails + Cron-Pfade (meeting-briefing, automation-runner) — nur wenn diese sendEmailWithTracking MIT ownerUserId aufrufen. Code-Audit-Followup empfohlen.

## Fix

**MIG-040** angelegt + auf Production appliziert via SSH+base64:

```sql
ALTER TABLE emails
  ADD COLUMN IF NOT EXISTS owner_user_id UUID
    REFERENCES profiles(id) ON DELETE SET NULL;

CREATE INDEX IF NOT EXISTS idx_emails_owner_user_id
  ON emails(owner_user_id);

NOTIFY pgrst, 'reload schema';
```

Plus Verify-Block der per `information_schema.columns` checkt + `RAISE EXCEPTION` triggert wenn Spalte fehlt.
