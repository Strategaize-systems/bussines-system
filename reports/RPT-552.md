## Wiring-Verification

```
HelpGuidePage (Server)
  ├── readFile(filePath) → markdown
  ├── renderLegalMarkdown(markdown) → html
  └── loadHotspotPage(slug) → hotspotData (oder null bei ENOENT)
       │
       └── parseHotspotPageJson(raw, slug) [throw bei Drift]
            └── HotspotPageSchema.parse + slug-match + id-uniqueness
       └── renderLegalMarkdown(hotspot.body_md) → bodyHtml (pre-render)
  →
  HelpPageShell ({ html, imageBlock })
    ├── (imageBlock ? <div className="mb-8">{imageBlock}</div> : kein Wrapper)
    └── <article className="help-content" dangerouslySetInnerHTML={{__html: html}} />

  imageBlock = <HotspotImage data={hotspotData} /> (Client)
    ├── State: openHotspotId
    ├── <figure>
    │    ├── <img src/alt/width/height/loading=lazy>
    │    └── <div className="hidden md:block"> [Desktop overlays]
    │         └── data.hotspots.map(h => <button left:%/top:%/w:%/h:% onClick={setOpenHotspotId(h.id)}>)
    ├── <ol className="md:hidden"> [Mobile list]
    │    └── data.hotspots.map(h => <li><strong>{title}</strong> + <div dangerouslySetInnerHTML={bodyHtml}>)
    └── <HotspotModal hotspot={openHotspot ?? null} open={openHotspotId !== null} onOpenChange>
         └── Base UI <Dialog> + <DialogContent max-h-[80vh] overflow-y-auto>
              ├── <DialogTitle>
              ├── <div dangerouslySetInnerHTML={bodyHtml}> [.help-content CSS-Reuse]
              └── optional <video controls src={videoUrl}>
```

Wiring **clean**:
- Server → Client Props sind nur primitive + Plain-Object-Array (`feedback_rsc_no_function_props` eingehalten)
- Single-Instance Modal-State (kein Stacking-Bug)
- Pre-rendered bodyHtml (kein Client-Side-Markdown-Parsing)
- ENOENT-Fallback transparent (Backward-Compat ohne Code-Branch in page.tsx)
- Responsive via Tailwind-only (kein Hydration-Mismatch-Risiko)
