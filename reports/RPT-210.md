## Code-Review Highlights

**Pattern-Konsistenz mit OpenAI-Adapter:**
- Response-Format-Switch (text/srt/vtt → string, verbose_json → text+language+duration, json → text)
- Error-Mapping (timeout-shaped Error → spezifische Timeout-Message, sonst generischer Fehler)
- Sprach-Default `de`, Timeout-Default 120s

**Azure-spezifische Ergaenzungen:**
- Config-Objekt-Pattern statt nackter Process-Env-Reads (testbar, validierbar)
- `clientFactory` injectierbar fuer Tests (clean dependency injection)
- Audit-Log mit Region-Extraktion aus Endpoint-URL
- Strukturierter Konfig-Fehler vor Client-Konstruktion

**Achtung-Bereiche (akzeptiert):**
- Audit-Log emittiert Plaintext-`console.info`, nicht in DB. Fuer V5.2 ausreichend (data-residency.md-Regel verlangt Audit-Trail, nicht zwingend DB-persistiert).
- `model: config.deployment` — Azure nutzt Deployment-IDs, dieser Wert wird im API-Aufruf als Model-Parameter mitgegeben. Falls Azure das anders erwartet, wird das beim Pre-Go-Live-Switch sichtbar (Risk dokumentiert).
