## Wiring Verification — Branding-Form-Chain

```
ConditionalColorPicker (Toggle/Color-Input)
    ↓ onChange(string | null)
BrandingForm (primaryColor state: string | null)
    ↓ handleSubmit: formData.set("primary_color", primaryColor ?? "")
form action={handleSubmit}
    ↓ FormData
updateBranding (Server Action, actions.ts)
    ↓ sanitizeColor(formData.get("primary_color"))
PostgreSQL branding_settings.primary_color
    ↓ getBranding / getBrandingForSend
renderBrandedHtml(body, branding)
    ↓ isBrandingEmpty? Ja → textToHtml(resolvedBody) [V5.2-bit-identisch]
                       Nein → branded HTML mit b.primaryColor || DEFAULT
```

**Verifiziert:**
- ✅ Component → State: `onChange(null)` setzt parent state auf null
- ✅ State → FormData: `primaryColor ?? ""` korrekt mappt null zu empty-string
- ✅ FormData → DB: `sanitizeColor` mappt empty-string zu null (kein Bypass)
- ✅ DB → Renderer: `isBrandingEmpty` checkt nicht nur Farben, sondern alle Branding-Felder
- ✅ Renderer-Empty-Fallback: Ruft `textToHtml(resolvedBody)` direkt auf, ohne Branded-Template

## Findings

### Medium

#### M1 — ConditionalColorPicker selbst nutzt useEffect-set-state-Pattern
**Datei:** [cockpit/src/components/branding/conditional-color-picker.tsx:39-41](cockpit/src/components/branding/conditional-color-picker.tsx#L39-L41)

```ts
useEffect(() => {
  setEnabled(value !== null);
}, [value]);
```

Genau das Pattern, das MT-3 in den 2 E-Mail-Dialogen entfernt hat (`react-hooks/set-state-in-effect`). Lint flagt es nicht, vermutlich weil React 19 die "External-Sync-of-Single-Prop-Mirror"-Variante als zulaessig wertet.

**Empfehlung (Refactor zu derived-state):**

```ts
const enabled = value !== null;

const handleToggle = (next: boolean) => {
  onChange(next ? defaultColor : null);
};
```

Cleaner React-Idiom (state-derivation aus props ohne lokales State-Mirror). Verhalten identisch — beim Toggle-Click wird onChange aufgerufen, parent setzt state, derived `enabled` re-derived. Keine ueberfluessige `setEnabled`-Doppel-Triggerung.

**Klassifikation:** Medium — kein funktionaler Bug, kein Lint-Befund. Nur Pattern-Inkonsistenz mit dem MT-3-Spirit. Kann in V5.4-Patch oder spaeter gefixt werden. **Nicht V5.4-Release-blockierend.**
