## ISSUE-080 Discovery + Live-Fix

Waehrend Live-Smoke S2 entdeckt: Public-Route `/p/strategaize/datenschutz` returnt HTTP 404 trotz existing DB-Row + content_md=10205.

**Root Cause:** SLC-841 Phase 2 Backfill (SQL `lower(replace(name, 'ä','ae'), ...)`) hat Team "Strategaize" → Slug `strategaize` gesetzt. **`strategaize`** ist aber in `reserved-slugs.ts` Strategaize-Common-Reserved-Liste. Page.tsx Zeile 21 `isReservedSlug(tenantSlug) → true → notFound()`.

**Live-Fix 2026-05-23:**
```sql
UPDATE teams SET slug='strategaize-transition-bv' WHERE slug='strategaize';
NOTIFY pgrst, 'reload schema';
```

Nach Fix: `/p/strategaize-transition-bv/datenschutz` HTTP 200 + Render-Verify mit Auftragsverarbeiter-Tabelle.

**Issue dokumentiert** als `ISSUE-080` (Severity Medium) in KNOWN_ISSUES.md.
